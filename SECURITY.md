# Security Policy

Author: André Henrique (@mrhenrike) | União Geek — https://github.com/Uniao-Geek

## Supported Scope

RouterXPL-Forge scope for this repository:

- In scope: routers, switches, taps, fw and ngfw.
- Out of scope: camera, printer, dvr modules.

## Reporting a Vulnerability

Please use GitHub private vulnerability reporting for this repository (Security -> Report a vulnerability).

Include:

- affected module path and version/commit
- reproducible steps
- expected impact and observed behavior
- logs or tracebacks (with sensitive data removed)

## Response Targets

- Acknowledgement target: 72 hours
- Initial triage target: 7 business days
- Mitigation/fix timeline: depends on severity and reproducibility

## Safe Testing Rules

- Test only on assets you own or are explicitly authorized to assess.
- Do not submit weaponized malware or destructive payloads.
- Prefer reproducible, minimally invasive PoCs.
- Respect all applicable laws and organizational policies.

## Disclosure

Coordinated disclosure is preferred. Public disclosure before maintainer triage may be rejected from governance workflows.
